# 阶段2实现报告：SDK 运行时和本地 Bridge

## 阶段目标

让模板运行时可以注册函数、发起函数调用、接收函数调用结果，并让本地 Electron Bridge 可以在模板页和网关 TCP 通道之间转发函数协议。

## 已实现内容

1. SDK 新增通信层：
   - `LocalBridgeWebSocketTransport`
   - `RuntimeTemplateBridge`
   - 默认全局 Bridge 实例

2. SDK 新增模板作者 API：
   - `useTemplateBridge`
   - `registerTemplateFunction`
   - `unregisterTemplateFunction`
   - `invokeTemplateFunction`
   - `emitTemplateEvent`
   - `onTemplateEvent`

3. SDK 新增通信类型：
   - `TemplateBridge`
   - `TemplateFunctionInvokeTarget`
   - `TemplateFunctionInvokeOptions`
   - `TemplateFunctionHandler`
   - `TemplateFunctionResult`
   - `TemplateEventTarget`
   - `TemplateEventHandler`

4. 本地 Bridge 新增协议支持：
   - `register_functions`：模板页向 Bridge 注册当前页面已实现的函数。
   - `invoke_function`：模板页发起函数调用。
   - `template_function_call`：Bridge 下发函数调用给目标模板页。
   - `function_result`：目标模板页返回执行结果。
   - `function_error`：目标模板页返回执行错误。
   - `template_function_result`：Bridge 将函数结果回传给网关。

5. 本地 Bridge 支持同设备本地调用：
   - A 模板页用 `target.mode = local` 调用。
   - Bridge 按已注册函数列表找到 B/N 模板页。
   - 目标模板执行 handler 后结果回到发起方。

6. 兼容旧模板入口：
   - `gateway/electron/template/exhibition-sdk.js` 新增同名函数 API。
   - `client/web/exhibition-sdk.js` 新增同名函数 API，保证模板真实加载入口可用。
   - 旧的 `emit/on/off/onReady` 行为保持不变。

## 已知边界

1. 当前阶段已经完成本地 Bridge 闭环，但跨设备 `binding/unicast/multicast/broadcast` 仍需要网关路由支持。
2. 当前参数校验主要发生在函数声明阶段，运行时 handler 入参暂未按 manifest 做强校验。
3. 函数调用日志、超时聚合、跨设备结果聚合将在后续网关路由阶段继续实现。
