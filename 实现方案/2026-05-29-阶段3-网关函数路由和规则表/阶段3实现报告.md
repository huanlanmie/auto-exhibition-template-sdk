# 阶段3实现报告：网关函数路由和规则表

## 阶段目标

让网关能够接收模板函数调用，并按直接目标或绑定规则转发给其他设备模板函数。

## 已实现内容

1. 网关本地数据库新增表：
   - `function_binding_rules`：保存最小函数绑定规则。
   - `function_call_logs`：保存函数调用发送、成功、失败记录。

2. 网关数据库模块新增方法：
   - `upsertFunctionBindingRules`
   - `getFunctionBindingRulesForSource`
   - `getAllFunctionBindingRules`
   - `insertFunctionCallLog`
   - `getFunctionCallLogs`

3. 网关主进程新增函数路由能力：
   - `template_function_call`
   - `template_function_result`
   - `target.mode = binding`
   - `target.mode = unicast`
   - `target.mode = multicast`
   - `target.mode = broadcast`

4. `binding` 规则支持：
   - 按 `source_client_id + source_function_name` 命中规则。
   - `source_client_id` 为空时作为通用规则。
   - `target_json` 支持多个模板 endpoint。
   - `args_mapping_json` 支持：
     - 常量：`{ "targetParam": "value", "value": 80 }`
     - 源参数：`{ "targetParam": "id", "source": "args.id" }`
     - 上下文：`{ "targetParam": "fromClientId", "source": "context.from.clientId" }`

5. 网关新增 IPC：
   - `function-binding:get-rules`
   - `function-binding:upsert-rules`
   - `function-binding:get-call-logs`

6. 管理后台仓库新增 SQL 升级脚本：
   - `template_function_manifest`
   - `device_link_group`
   - `device_link_group_member`
   - `function_binding_rule`
   - `function_call_log`
