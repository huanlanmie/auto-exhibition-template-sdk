# 阶段4实现报告：硬件能力扩展骨架

## 阶段目标

让模板函数联动模型可以自然扩展到硬件，不把串口、MQTT、HTTP 等具体协议写死在 SDK 或路由逻辑里。

## 已实现内容

1. 网关新增硬件适配器注册表：
   - `HardwareAdapterRegistry`
   - `register`
   - `unregister`
   - `invoke`
   - `getManifests`
   - `onEvent`

2. 新增 mock 硬件适配器：
   - `hardwareId = mock-hardware`
   - `function = echo`
   - 可用于验证模板函数 -> 硬件函数的调用链路。

3. 函数规则目标支持硬件 endpoint：

   ```json
   {
     "endpointType": "hardware",
     "hardwareId": "mock-hardware",
     "functionName": "echo"
   }
   ```

4. 硬件事件预留入口：
   - adapter 可以通过 `onEvent` 上报事件。
   - 网关把硬件事件转成统一函数触发源。
   - 可按 `source_client_id = hardware:{hardwareId}` 和 `source_function_name = eventName` 命中绑定规则。

5. 网关新增 IPC：
   - `hardware:get-manifests`
