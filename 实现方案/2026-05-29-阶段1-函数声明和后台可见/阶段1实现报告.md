# 阶段1实现报告：函数声明和后台可见

## 阶段目标

让模板可以通过 `configJson.functions` 正式声明函数能力，并让 SDK 构建产物与管理后台能够读取和展示这些函数清单。

## 已实现内容

1. SDK 新增模板函数声明类型：
   - `TemplateFunctionDefinition`
   - `TemplateFunctionParam`
   - `TemplateFunctionManifest`
   - `TemplateSdkValueMapMeta`

2. SDK 新增函数声明校验：
   - `functions` 必须是对象。
   - 函数名限制为字母、数字、`_`、`$`、`.`、`-`，最长 64 位，且必须以字母、`_` 或 `$` 开头。
   - `direction` 支持 `in`、`out`、`inout`。
   - 参数类型支持 `string`、`number`、`boolean`、`object`、`array`、`any`。
   - `transport.timeoutMs` 限制在 100 到 30000。
   - `transport.qos` 支持 `at_most_once`、`at_least_once`。
   - `dataSchema.fields[].key` 禁止使用 SDK 保留字段 `__templateSdk`。

3. SDK 构建 `valueMap` 时写入保留元信息：

   ```json
   {
     "__templateSdk": {
       "schemaVersion": 1,
       "functions": {}
     }
   }
   ```

4. SDK 新增独立构建函数：
   - `buildTemplateFunctionManifest`
   - `buildTemplateSdkMeta`

5. 管理后台 `TemplateAnalysis` 新增函数能力展示：
   - 解析 `templateConfig.functions`。
   - 展示函数名、方向、显示名、说明、参数、默认值。
   - 发布模板时把函数清单同步写入发布后的 `valueMap.__templateSdk.functions`。
   - 预览运行时同时兼容新路径 `assets/auto-exhibition-template-sdk/valueMap.json` 和旧路径 `assets/template-sdk/valueMap.json`。
